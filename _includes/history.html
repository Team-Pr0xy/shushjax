Evolution

shushjax is based on several similar libraries

Original Inspiration

PJAX

Released in in February 2011 by Chris Wanstrath (defunkt), PJAX introduced the world to the concept of injecting new content into existing pages. While this could already be done with some javascript frontend frameworks, they require a lot of implementation work and are geared more towards webapps, not sites. In addition, PJAX took advantage of the new pushstate history APIs to enable real permanlinks and history without ugly hacks. PJAX has grown to become one of the most starred repositories on GitHub, and is even used on GitHub itself. Despite this, PJAX is limited by its requirement for a special backend and use of jQuery.

Ditching jQuery

PJAX-Standalone

Loosely based on the original PJAX, PJAX-Standalone by Carl Saggs (thybag) is a lighter implementation that doesn't depend on jQuery. shushjax is almost entirely based on PJAX-standalone.

No more backend

PJAX-FW

Inspired by PJAX, pjax-fw by Joah Gerstenberg (JoahG) is a fully frontend framework. It is where the concept of shushjax originated, although none of its code was used.
