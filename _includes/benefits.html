Benefits

Using shushjax is as simple as adding a javascript file to your site. For even more benefits, optionally use partial file support

Dramatically reduce page load times

shushjax uses the CSS, Javascript, webfonts, and other resources that your visitors already have loaded. The only server requests required when navigating to a different page are for the new content. Additionally, CSS and Javascript don't need to be re-processed, leading to a major performance boost on slow clients and mobile devices.

No external dependencies

Some other solutions rely on the jQuery library. This adds on a large bandwidth and processing burden, especially on mobile devices. We believe that a program designed to make things faster shouldn't depend on components that slow it down. shushjax is standalone, not relying on loading any other libraries.

No special backends required

Originally, PJAX used a special request header to get partial content from the server. This meant that custom application code was required, which limited where PJAX could be used and caused caching problems. shushjax doesn't rely on special headers or dynamic server code. You can take the full power of shushjax just about anywhere, including static hosts like Amazon S3 and GitHub Pages.
