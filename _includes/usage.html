Quick Start

Get ready

Before you begin

Your site needs to use the same CSS, javascript, and font resources on every page in order for shushjax to function correctly. These resources are re-used when navigating between pages.

Add the code

Insert a link to shushjax in your <head>

Get the shushjax script and place it on your webserver, then add a link in the <head> of your html, like this: 


<html>
<head>
...
<script src="/shushjax.min.js"></script>
...
</head>
<body>
...
</body>
</html>

Serve minified and gzipped for maximum performance

{% include cdn.html %}

Configure content

Tell shushjax what content to load

shushjax will only replace content that you configure. You'll need a 'container' <div> on each page, which is where shushjax will insert content from new pages. Immediately below the link to the shushjax javascript file, initialize shushjax and specify the container and any options. 


<html>
<head>
...
<script src="/shushjax.min.js"></script>
<script>shushjax.connect({'container': 'content',});</script>
...
</head>
<body>
...
</body>
</html>

Go Faster

Optional: Use partial file support

For even faster performance, deliver partial pages to shushjax. Partial pages contain only the contents of the container div, saving bandwidth.
When partial page support is enabled, shushjax looks for partial pages under the /partials directory. For example, if the original page is located at 

http://example.com/oceans/pacific.html
shushjax will look for the partial version at 
http://example.com/partials/oceans/pacific.html
Once you've set up the files on your server, enable partial pages in your shushjax.connect 
<script>shushjax.connect({'container': 'content', 'partial': true,});</script>

Keep your original files

The original files will still be used the first time that a browser loads a page on your site and are also used by any clients, such as search engines, that are incompatable with shushjax.

Full Documentation

Core Concepts

It's just ajax and pushState

shushjax is designed to make your pages load crazy fast by only loading new content and injecting it into the current CSS and layout. shushjax uses ajax and pushState to deliver a fast browsing experience with real permalinks, page titles, and a working back button. shushjax works by grabbing html from your server via ajax and replacing the content of a container on your page with the ajax'd html. It then updates the browser's current url using pushState without reloading your page's layout or any resources (js, css, fonts), giving the appearance of a fast, full page load. While AJAX-driven webapps have been around for years using some frameworks, implementing them has been complicated. By contrast, shushjax is designed as a simple drop-in solution.

Caveats

Get started right

Using shushjax dramatically alters the way that pages and resources are loaded on your website. If something is misconfigured, breakage could occur. Remember that when navigating to a new page with shushjax, only new resources in the new container <div> will be loaded. All of the resources in other parts of the new page's HTML, such as CSS and javascript in the <head>, will not be loaded. All of the resources outside of the container <div> must remain constant across all pages of a shushjax-enabled site.

Debugging

shushjax automatically delivers verbose logs of everything it does to your browser's javascript console. You can see which links it does and does not activate on, as well as the page loading process and fired events.
Types of console messages used by shushjax:

  • Error something is broken
  • Info a fallback was activated or something unexpected happened, but shushjax recovered seamlessly
  • log everything that happens inside shushjax

Other rules

When processing the links on a page, shushjax will only activate itself on links that point to exactly the same host. The URL scheme, hostname, and port must be the same as the current page, otherwise the link will load regularly.

Compatibility

Browser support

Nearly all browsers behave as expected with shushjax, and the vast majority are able to use the performance benefits. shushjax is designed with a series of graceful fallbacks. When a browser doesn't support part of shushjax, the code disables itself and pages load normally.

Fallbacks

  • No Javascript links load normally, shushjax doesn't activate
  • No pushState links load normally, shushjax pushes the browser to it directly
  • No URL() partial file support is disabled, whole HTML files are loaded instead
  • No XHR2 the classic XMLHttpRequest is used instead
These fallbacks enable shushjax to consistently deliver a fast, working browsing experience with real permalinks, page titles, and a working back button, no matter the browser.

Explanation of support terms

  • Supported shushjax works as intended
  • Limited partial file support Partial files have limited support, but browsers will automatically fall back to loading whole pages if needed.
    • webkitURL() Some browsers implement URL() as webkitURL(). shushjax automatically detects and uses it instead.
    • no URL() support Making the partial file request requires that the browser supports URL(). If not, partial file support will be automatically disabled.
  • Seamless fallback The functions used by shushjax are unsupported, but shushjax automatically disables itself and links load like they would without shushjax.
  • Unsupported This browser may or may not work, but it receives no testing or bugfixes.
  • Old versions of Internet Explorer Along with Windows XP, Internet Explorer versions 6, 7, and 8 are end-of-life as of April 8, 2014. At this time, these browsers will be officially unsupported. Although the code for them will remain and they may continue to work, they will not receive testing or bugfixes.
  • Broken shushjax causes severe breakage

Setup

Installation options

Insert a link to shushjax in your <head>

No matter how you use shushjax, you'll need to get the script and place it on your webserver, then add a link in the <head> of your html: 

<script src="/shushjax.min.js"></script>

Serve minified and gzipped for maximum performance

{% include cdn.html %}

Tell shushjax what content to load

shushjax will only replace content that you configure. You'll need a 'container' <div> on each page, which is where shushjax will insert content from new pages. Immediately below the link to the shushjax javascript file, initialize shushjax and specify options. 

<script>shushjax.connect();</script>
When enabling shushjax, you can choose to activate on all normal links or just links with a special class. You can also specify either one container for all loaded content or use special settings on individual links.

The container

This is where freshly loaded content gets inserted.
Use the global container setting in your shushjax.connect: 

<script>shushjax.connect({container: 'content',});</script>
And/ or set each link individually with the data-shushjax attribute: 
<a href='unicorns.html' data-shushjax='content'>Unicorns!</a>

Using classes

You can choose to only activate links with a specific class with the useClass setting: 

<script>shushjax.connect({container: 'content', useClass: 'shushjax-enabled',});</script>

Options

Further configuration

shushjax has several options that you can specify during the initialization call, like this 

<script>shushjax.connect({options});</script>
  • container string The <div> where freshly loaded content gets inserted
    • container: 'content',
  • smartLoad true|false Ensure that the returned HTML is usable. Enabled by default. You can disable this for a slight performance boost if you are absolutely sure that the returned HTML is a properly formatted partial file. It's suggested that you keep this enabled so that shushjax can automatically work around problems with the returned page. 
    smartLoad: true,
  • useClass string Only activate shushjax on links with the provided class. Using this can make implementing shushjax easier on complex sites. 
    useClass: 'shushjax-enabled',
  • partial true|false Use partial file support. Disabled by default. When enabled, shushjax will load pages using partial files located under the /partials directory. 
    partial: true,
  • parseLinksOnload true|false Make links on loaded pages use shushjax. Enabled by default. 
    parseLinksOnload: true,

Use Partial Files

Optional: Use partial file support

You don't need to make any changes to your backend to benefit from shushjax. If you are unable to change the backend code, or simply do not want to, so long as smartLoad is enabled (which it is by default), shushjax will extract the container_divs content from the returned HTML and apply it to the current page meaning shushjax loading will still work as expected (although some of the performance gains may be lost). Callbacks

Available callbacks

shushjax implements the following callbacks:

  • beforeSend - Called before the ajax request is made
  • complete - When the ajax request has completed
  • success - When the ajax request has completed successfully. This is useful for analytics.
  • requestError - When the ajax request did not complete successfully (error 500, etc)
  • generalError - When a problem occurs elsewhere, like parsing the document
The callbacks are specified as part of the original shushjax.connect method: 
shushjax.connect({
    container: 'content',
    beforeSend: function(){console.log("before send");},
    complete: function(){console.log("done!");},
});

Programmatic Usage

Invoke shushjax yourself

You can invoke a shushjax page load programmatically by calling the shushjax.invoke() method. At minimum the invoke method must be given a url and container attribute. It can also be provided with a title, parseLinksOnload setting and any callbacks you wish to use. 

shushjax.invoke({url:'page1.php', container: 'content',});
