Quick Start

Get ready

Before you begin

Your site needs to use the same CSS, javascript, and font resources on every page in order for shushjax to function correctly. These resources are re-used when navigating between pages.

Add the code

Insert a link to shushjax in your <head>

Place the shushjax script on your webserver, then add a link in the <head> of your html, like this: 


<html>
<head>
...
<script src="/shushjax.js"></script>
...
</head>
<body>
...
</body>
</html>

Coming Soon: Use a CDN

Configure content

Tell shushjax what content to load

shushjax will only replace content that you configure. Immediately below the link to the shushjax javascript file, initialize shushjax and specify options. 


<html>
<head>
...
<script src="/shushjax.js"></script>
<script>shushjax.connect({'container': 'content',});</script>
...
</head>
<body>
...
</body>
</html>

Compatability

Browser support

Nearly all browsers behave as expected with shushjax, and the vast majority are able to use the performance benefits. shushjax is designed with a series of graceful fallbacks. When a browser doesn't support part of shushjax, the code disables itself and pages load normally.

Explanation of support terms

  • Supported shushjax works as intended
  • Limited partial file support Partial files have limited support, but browsers will automatically fall back to loading whole pages if needed.
    • webkitURL() Some browsers implement URL() as webkitURL(). shushjax automatically detects and uses it instead.
    • no URL() support Making the partial file request requires that the browser supports URL(). If not, partial file support will be automatically disabled.
  • Seamless fallback The functions used by shushjax are unsupported, but shushjax automatically disables itself and links load like they would without shushjax.
  • Unsupported This browser may or may not work, but it receives no testing or bugfixes.
  • Old versions of Internet Explorer Along with Windows XP, Internet Explorer versions 6, 7, and 8 are end-of-life as of April 8, 2014. At this time, these browsers will be officially unsupported. Although the code for them will remain and they may continue to work, they will not receive testing or bugfixes.
  • Broken shushjax causes severe breakage

Options

Further configuration

shushjax has several options that you can specify during the initialization call, like this 

<script>shushjax.connect({options});</script>
  • smartLoad true|false Ensure that the returned HTML is usable. Enabled by default. You can disable this for a slight performance boost if you are absolutely sure that the returned HTML is a properly formatted partial file. It's suggested that you keep this enabled so that shushjax can automatically work around problems with the returned page. 
    'smartLoad': true,
  • useClass string Only activate shushjax on links with the provided class. Using this can make implementing shushjax easier on complex sites. 
    'useClass': 'shushjax-enabled',
  • partial true|false Use partial file support. Disabled by default. When enabled, shushjax will load pages using partial files located under the /partials directory. 
    'partial': true,
  • parseLinksOnload true|false Make links on loaded pages use shushjax. Enabled by default. 
    'parseLinksOnload': true,

Callbacks

Available callbacks

shushjax impliments the following callbacks:

  • beforeSend - Called before ajax request is made
  • complete - When ajax request has completed
  • success - When ajax request has completed successfully
  • error - When ajax request did not complet successfully (error 404/500 etc)
The callbacks are specified as part of the original shushjax.connect method: 
shushjax.connect({
    'container': 'content',
    'beforeSend': function(){console.log("before send");},
    'complete': function(){console.log("done!");},
});
